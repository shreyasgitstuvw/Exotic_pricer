# P2 Benchmark Protocol (locked before experiments — Orion discipline)

## Baselines
1. Static Heston, calibrated once per {day, week, month} — the "unrealistic assumptions" strawman must be steelmanned: report all three recalibration frequencies.
2. Black-Scholes flat-vol (sanity floor).
3. Full optimizer calibration (DE→L-BFGS-B) = accuracy ceiling for the M2 surrogate.

## Metrics
- Vanilla repricing: IV RMSE (bps) on held-out surfaces; report by moneyness bucket and tenor.
- Calibration speed: wall-time per surface, optimizer vs surrogate (same hardware, median of 20 runs).
- Exotic price gaps: (model_b − model_a)/vega-equivalent, per instrument, with MC standard errors ≤ 1/10 of the gap being claimed.
- Hedging: daily delta-hedge PnL variance + p5 tail on 500 bootstrap path resamples; both models, same paths.

## Splits & hygiene
- Train/val/test by TIME (no shuffling surfaces). Test = final 3 months, untouched until M2 gate.
- Every result table auto-generated by `experiments/run_*.py` scripts committed with results — no hand-edited numbers (same rule as the Orion engine's metrics writer).
- Seeds fixed and logged; MC path counts stated on every figure.

## Kill criteria
- If surrogate can't beat 25 bps on val by end of W6 → ship M1 + M3-under-static-Heston as the honest result and post the negative finding. A defended negative beats an oversold positive (you have the 0-for-7 graveyard as precedent).
