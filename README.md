# P2 — Heston + neural components (data layer & scaffold)

Flagship research project. See `PLAN.md` (milestones/gates), `BENCHMARKS.md` (locked metrics), and
`DECISIONS.md` (pre-registered design log). This README covers the **M0 data layer** built so far.

## What exists now (M0, short-end layer)

A config-driven pipeline that turns the raw 1-minute NIFTY option strips into clean, arbitrage-checked
**EOD short-dated smiles**, with a manifest of provenance/quality per day.

```
src/hestonnn/
  config.py            # YAML-driven config (no hand-tuned numbers in code)
  data/
    loader.py          # discover + load weekly parquet strips, futures, VIX
    expiry.py          # EMPIRICAL expiry detection via ATM time-value collapse (D3)
    forward.py         # forward F + discount factor implied from put-call parity
    iv.py              # Black-76 pricing + vectorized IV inversion (recomputed, D1)
    surface.py         # per-date close-window smile assembly + hygiene
    arbitrage.py       # intra-smile butterfly + vertical no-arb flags
    bhavcopy.py        # INTERFACE STUB for the EOD multi-expiry layer (D3-A) — see below
experiments/
  build_surfaces.py    # entry point: strips -> clean smiles + manifest (the only writer of results)
configs/data.yaml      # all paths, close window, thresholds, hygiene bounds
tests/                 # IV round-trip, PCP forward recovery, expiry-shift, arb flags
```

### Run it
```
pip install -e .            # or: pip install numpy pandas pyarrow scipy pyyaml
python experiments/build_surfaces.py --config configs/data.yaml --years 2024
# sandbox: add --data-root <mount path> --out-dir data

```

## Two-layer data model (DECISIONS D3, Option A)

The `HISTORY_NIFTY` data is **front-contract only** (maturities ≤ ~22 days), so it cannot identify
full Heston (κ, θ need a term structure) or price multi-year autocallables. Resolution:

- **Short-end / high-res** — this 1-min strip → M3 hedging sim + short-smile study. **Done (above).**
- **Long-end / full surface** — NSE **EOD bhavcopy** (all listed expiries) → the primary M1/M2
  calibration surface and long-dated vol for M3. **Pending your data.**

### Bhavcopy layer — built & validated
`data/bhavcopy.py` parses BOTH NSE schemas (new UDiFF + legacy) via header auto-detection;
`data/surface_bhav.py` assembles the maturity×moneyness surface through the same forward/iv/arbitrage
machinery. One file gives the full term structure (validated on 2026-07-10: liquid tenors 4→~900d).

**Auto-download from NSE (runs on YOUR machine — needs `requests` + NSE-reachable network):**
```python
from hestonnn.data.sampling import schedule
from hestonnn.data.fetch_bhavcopy import fetch_many
dates = schedule("2024-08-01", "2026-07-10", weekly_weekday=2, monthly_day=15)  # ~sampled set
fetch_many(dates, cache_dir="bhavcopy")     # cookie-primed, retries, caches, skips 404 holidays
```
Then build surfaces (fetches missing dates, or `--no-fetch` for cached-only):
```
python experiments/build_bhavcopy_surfaces.py --config configs/data.yaml \
    --start 2024-08-01 --end 2026-07-10 --weekly-weekday 2 --cache-dir bhavcopy --out-dir data/bhav
```
The downloader picks the UDiFF vs legacy URL automatically by date (cutover 2024-07-08). You don't
need every trading day — the sampler's weekly+monthly+event schedule keeps the file count small.

### Calibration + M1 instability
`heston_ref.py` (reference Heston CF pricer, validated to ~0.5 bps) + `calibrate.py` (vega-weighted
IV RMSE, DE→L-BFGS-B). First real fit on 2026-07-10 bhavcopy: **34 bps** vega-weighted (belly 12–22
bps; 4d strains at 77 bps — the M2 motivation). See `data/first_fit_20260710.txt`.

`experiments/run_m1_instability.py` builds the parameter time series + money-plot:
```
python experiments/run_m1_instability.py --source short-end --every 8      # 1016-day short-end proxy
# FULL 5-param M1 on your downloaded bhavcopy set (run on your machine — thorough DE per date):
python experiments/run_m1_instability.py --source bhavcopy --cache-dir bhavcopy \
    --config configs/data.yaml --out-dir data/m1
#   resumable (appends per date, skips done); ~10-15s/date x ~110 dates. Add --every 2 to subsample.
```
The **short-end proxy** (constrained Heston: κ,θ fixed; σ_v,ρ,v0 free) runs on the 1-min smiles now
and produces `money_plot_shortend.png` + `jump_stats_shortend.txt`. It shows σ_v/ρ swinging hard —
part genuine regime moves, part single-short-smile identifiability noise, which is itself the
argument for the full-surface bhavcopy M1 and the M2 neural layer. The observed ATM-IV panel is the
clean signal (NIFTY vol 2021 highs → 2025 lows with spikes).
